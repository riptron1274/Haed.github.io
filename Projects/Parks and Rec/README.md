## Overview
This project demonstrates end-to-end SQL skills using a fictional
Parks & Recreation employee dataset. The database was designed and
populated from scratch and used to perform analytical queries that
mirror real-world HR and organizational analysis tasks.

The goal of the project is to showcase practical SQL proficiency,
from basic querying to advanced database concepts.

---

## Dataset
The database consists of three main tables:

- **employee_demographics**  
  Employee age, gender, and birth date

- **employee_salary**  
  Job roles, salaries, and department assignments

- **parks_departments**  
  Organizational department structure

All data was manually created to simulate realistic organizational data.

## Key SQL Skills Demonstrated

### Core SQL
- SELECT, WHERE, ORDER BY, LIMIT
- Filtering with conditions and pattern matching
- Aggregations and GROUP BY
- HAVING clauses

### Joins & Relationships
- INNER JOIN
- RIGHT JOIN
- Multi-table joins

### Advanced SQL
- Subqueries
- CASE statements
- UNION
- Window functions (`ROW_NUMBER`, `RANK`, `DENSE_RANK`)
- Common Table Expressions (CTEs)

### Database Features
- Temporary tables
- Stored procedures
- Triggers
- Scheduled events
